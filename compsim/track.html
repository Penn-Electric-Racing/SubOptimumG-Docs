track#

class Track#

Competition tracks are represented as lists of corners. Each corner is a 2-element array [radius (meters), length (decimeters)]. Both measurements are relative to the centerline. (e.g. a corner that has a radius of \(9.125~\text{m}\) and a centerline length of \(57.3~\text{m}\) will be [9.125, 573]).

Note

A corner radius larger than \(10000~\text{m}\) will be interpreted by the code as a straight.

Note

Internally, the representation for corner length gets converted from decimeters to number of simulation steps, where each step moves DISTANCE_STEP along the centerline. The conversion is done by applying a scaling \(\frac{0.1}{DISTANCE\_STEP}\) to the corner length, then rounding.

competition.yaml also stores a minimum time for each track, which helps quantify how well our car is doing compared to other cars (for now, the only available numbers are from Michigan 2023)

The Track class provides functionality to:

  • Create tracks from configuration dictionaries

  • Clean up corner definitions

  • Scale track dimensions

  • Compute event points based on lap times

Parameters

Parameters:

  • corners (list[list[float, float]]) – Ordered list of corner definitions, where each corner is [radius, length]

  • track_best_time (float) – Best achievable time for the track in seconds

  • worst_time_ratio (float) – Ratio defining the worst acceptable time relative to best time

  • min_points (float) – Minimum points that can be earned on the track

  • max_point_gain (float) – Maximum additional points that can be earned above min_points

  • scale_time_ratio (float) – Exponent used in points calculation to scale time differences

Methods#

classmethod Track.from_config(config)#

Creates a Track instance from a configuration dictionary.

Parameters:

config (dict) – Configuration dictionary containing track parameters

Returns:

A new Track instance

Return type:

Track

The configuration dictionary must contain all parameters required by the Track constructor, plus:

  • distance_step: Used to convert corner lengths to step counts

  • shorten: Additional scaling factor applied to corner lengths

Note

This method applies two scaling operations to the corner definitions:

  1. Converts corner lengths from decimeters to simulation steps

  2. Applies an additional scaling factor (currently used for Mich 2023 tracks)

The cleaned list has small corner radii increased to improve simulation behavior. This adjustment is currently used for Mich 2023 AutoX and Endurance tracks.

Track.compute_event_points(tyour)#

Computes points earned for a given lap time.

Parameters:

tyour (float) – Your lap time in seconds

Returns:

Points earned for the lap time

Return type:

float

Points are calculated using the formula:

\[points = min\_points + max\_point\_gain * \frac{(worst\_time/tyour)^{scale} - 1}{(worst\_time/best\_time)^{scale} - 1}\]

where:

  • worst_time = best_time * worst_time_ratio

  • scale = scale_time_ratio

Returns:

Modified list of corner definitions

For corners with radius ≤ 10m, the radius is scaled up using:

\[new\_radius = radius * ((20 - radius)/10)^{1.2}\]

Examples#

# Create accel track (a single corner with 100,000m radius)
config = {
    "corners": [[100000.0, 750.0]],
    "track_best_time": 3.642 # (Georgia Tech 2024)
    "worst_time_ratio": 1.5,
    "min_points": 4.5,
    "max_point_gain": 95.5,
    "scale_time_ratio": 1, # (no effect)
    "distance_step": 0.065, # (meters)
    "shorten": 1.0 # (unchanged)
}

track = Track.from_config(config)
